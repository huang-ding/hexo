启动时检查

Dubbo缺省会在启动时检查服务是否可用，不可用时抛出异常，阻止程序(spring)初始化完成

通过check="true/false"配置,可通过多种方式，spring配置文件，dubbo配置文件，jvm -D参数等

本地伪装

本地伪装通常用于服务降级，当服务提供者挂了，客户端不会抛出异常，通过Mock数据返回

示例

客户端提供Mock实现类，需和提供者实现同一个接口

return

可以使用return 返回一个字符串表示的对象

  • empty：空，基本类型默认值，集合的空值
  • null：null
  • true/false：true/false
  • JSON：反序列化JSON所得到的对象
throw

使用 throw来返回一个异常对象，作为mock的返回值

forec 和 fail

forec 强制使用mock行为，不走远程调用，fail与默认行为一直

负载均衡

在集群负载均衡时，dubbo提供了多种均衡策略，缺省为random随机

负载策略 LoadBalance

  • Random 随机
  • RoundRobin 轮询
  • LeastActive 最少活跃
  • ConsistentHash 一致性hash

负载均衡文档：负载

集群容错

在集群调度失败时，dubbo提供了多种容错方案，缺省为failover重试

模式

  • Failover 重试其他服务器
  • Failfast 快速失败，调用一次
  • Failsafe 安全失败
  • Failback 失败自动恢复，定时重发
  • Forking 并行调用，一个成功即可
  • Broadcast 广播所有，需要全部成功

集群文档：集群
